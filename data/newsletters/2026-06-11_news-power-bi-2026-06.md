## Le Power BI Update de juin 2026

Microsoft a publié le 2 juin son Feature Summary mensuel, doublé d’une vague d’annonces Build 2026 sur l’agentic stack. Le ton du mois : du pragmatisme. Pas une révolution, mais trois améliorations bien placées qui touchent directement le quotidien du report designer, un nouveau Date picker slicer (Preview), un premier stack agentique de reporting (Preview), et un polish tooltip (GA) qui a l’air anodin mais change vos habitudes.

### 1. Date picker for slicers (Preview)

Le “**Date picker for slicers** **(Preview)**”, est une nouvelle option pour sélectionner une période ou une seule date de votre calendrier en tant que filtre ! Il vous permet de poser une sélection **relative** (”Last 30 days”, “Year to date”) qui roule automatiquement à chaque refresh des données, sans rouvrir le rapport.

Sur un rapport avec 3 ou 4 slicers de date (un par page, un global, un pour la comparaison N-1…), vous économisez le rituel mensuel d’ouverture et le risque d’oubli. Vous supprimez aussi le besoin d’avoir 2 slicers par page (un “relatif” + un “absolu”), un seul contrôle fait les deux.

⚠️ Deux limites à connaître avant d’activer :

* **C’est une Preview**. À activer via \*Options > Preview features, à ne pas généraliser sur vos rapports business-critical tant que ce n’est pas GA. Testez sur un rapport interne d’abord.
* **Le jour de début de semaine n’est pas configurable**. Comme le filter pane historique, “Last Calendar Week” suppose une semaine \*\*Sunday-Saturday\*\*. Pour l’Europe (ISO Monday-Sunday), c’est un point bloquant, Microsoft a confirmé en commentaire travailler dessus, sans date annoncée.

Source : [Date Picker on Blog](https://community.fabric.microsoft.com/t5/Power-BI-Updates-Blog/Power-BI-June-2026-Feature-Summary/ba-p/5193264#toc-hId--1244838770)

### 2. AI-Powered Power BI reporting avec agent skills (Preview)

Microsoft pose une brique simple mais structurante : un **catalogue de skills spécialisés** pour Power BI, open source sur GitHub (`microsoft/skills-for-fabric`), consommable par votre agent IA préféré (GitHub Copilot CLI en cible principale, mais aussi Claude Code, Cursor, Windsurf, Codex).   
**L’idée** : en langage naturel, vous décrivez ce que vous voulez et un agent construit le dashboard. Au lieu d’un seul Copilot qui essaie de tout faire, vous avez **plusieurs skills** **spécialisés** qui savent chacun faire **une chose bien**.

Trois skills couvrent aujourd’hui l’authoring de rapport :

* **powerbi-report-planning** recueille les besoins. Quand vous dites ”crée-moi un dashboard”, ce skill vous pose 3-5 questions cadrées (audience, objectif, périmètre, livrable), inspecte votre semantic model, et produit un brief approuvable avant la moindre ligne de code.
* **powerbi-report-design** choisit le **quoi** : tone, archetype de page (Executive Summary, Operational Monitor, Analytical Canvas…), types de visuels, palette, layout, accessibilité.
* **powerbi-report-authoring** exécute le **comment** : crée et modifie les pages, visuels, filtres, thèmes dans un projet PBIP, valide le résultat, recharge Power BI Desktop et capture des screenshots pour vérifier le rendu.

#### Comment ça se passe ?

L’agent route automatiquement : un prompt “redesign cette page” déclenche `report-design`, un prompt "ajoute une page de drillthrough" déclenche `report-authoring`, un prompt "créons un nouveau rapport" déclenche `report-planning` qui appelle les deux autres derrière. Vous n’avez pas à savoir lequel choisir.

💡 **Soyez précis dans le prompt (audience, périmètre, décision visée, KPI…) !**

Source : [AI-Powered Power BI reporting — agent skills Preview](http://skills-for-fabric/plugins/powerbi-authoring/skills/powerbi-report-authoring/SKILL.md at main · microsoft/skills-for-fabric)

### 3. Tooltip options - sentence mode (GA)

Plus modeste, mais immédiatement actionnable. Dans le Format visual pane > General > Tooltips, vous avez désormais un mode “**sentence**” qui vous permet de construire une **phrase explicative en référant directement les champs du Tooltip field well**, sans créer de mesure DAX dédiée pour formater le texte.

Concrètement, au lieu d’écrire une mesure :

```
”Le chiffre d’affaires de “ & MAX(’Date’[Month]) & “ est de “ & FORMAT([Revenue], “€#,0”)
```

vous le composez directement dans le panneau de format.

✅ **La réalité du terrain :** ça paraît anecdotique, mais sur un rapport, vous économisez des dizaines de mesures cosmétiques qui polluaient votre modèle. Et surtout, vous donnez aux utilisateurs métier une explication en langage naturel **au hover**, sans devoir ouvrir un tooltip page complet.

Source : [Visual tooltips](https://learn.microsoft.com/fr-fr/power-bi/visuals/power-bi-visualization-tooltips-overview)
